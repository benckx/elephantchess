As I understand it, there is no quick fix. Only change for now is to add exponential backoff to the WS connection attempts and send a notification to the user when WS connection fails, so they know that they might not receive real-time updates and can try to fix it on their end (e.g. by disabling browser extensions or using a different network).

2025-12-06

  • Overhaul of the "move annotation symbols" system and the game analysis data. We now compare engine best move to the actually played move (which is actually what is required for proper Centipawn Loss (CPL) calculation), instead of comparing the previous position to the current position. We need to collect more analysis data for this to work and therefore, all games previously considered "analyzed" are now flagged as "partially analyzed". "Partially analyzed" games can be completed (so if you re-start analysis from there, it will start at around 50% completion).
